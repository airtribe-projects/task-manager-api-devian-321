# Task Manager API




## Overview

Welcome to the Task Manager API! This is a simple yet robust RESTful API built with Node.js and Express.js, designed to manage tasks. It allows users to create, retrieve, update, and delete tasks, with added functionality for filtering tasks by their completion status.

## Features

* **Create Tasks:** Add new tasks with a title, description, and completion status.
* **Retrieve Tasks:**
    * Fetch all tasks.
    * Fetch a single task by its unique ID.
    * **Filter Tasks:** Retrieve tasks based on their completion status (`completed=true` or `completed=false`).
* **Update Tasks:** Modify existing tasks (partial updates supported).
* **Delete Tasks:** Remove tasks by their ID.
* **Input Validation:** Ensures data integrity for task creation and updates.
* **Error Handling:** Provides meaningful error responses for invalid requests or non-existent resources.

## Setup Instructions

Follow these steps to get the Task Manager API up and running on your local machine.

### Prerequisites

* Node.js (LTS version recommended)
* npm (Node Package Manager, usually comes with Node.js)

### Installation

1.  **Clone the repository:**
    ```bash
    git clone <your-repository-url> 
    cd task-manager-api
    ```

2.  **Install dependencies:**
    ```bash
    npm install
    ```
    This will install Express.js and any other packages listed in `package.json`.

3.  **Create a `.env` file (optional but recommended):**
    If you're using `dotenv` for environment variables (like the `PORT`), create a `.env` file in the root directory of your project:
    ```
    PORT=3000
    ```
    Or, the application will default to port 3000.

4.  **Start the server:**
    ```bash
    node app.js
    ```
    You should see the message: `Server is listening on 3000` (or your specified port).

The API will now be running at `http://localhost:3000`.

## API Endpoints

This section documents all available API endpoints, their methods, expected request formats, and example responses. You can use tools like Postman, Insomnia, or `curl` to test these endpoints.

### Base URL

`http://localhost:3000`

---

### 1. Get All Tasks

* **Method:** `GET`
* **Path:** `/tasks`
* **Description:** Retrieves a list of all tasks.
* **Query Parameters:**
    * `completed`: (Optional) Filter tasks by their completion status.
        * `true`: Returns only completed tasks.
        * `false`: Returns only incomplete tasks.
        * **Validation:** If a value other than `true` or `false` is provided, a `400 Bad Request` error will be returned.
* **Example Request:**
    * `GET http://localhost:3000/tasks`
    * `GET http://localhost:3000/tasks?completed=true`
    * `GET http://localhost:3000/tasks?completed=false`
* **Example Response (200 OK - all tasks):**
    ```json
    [
      {
        "id": 1,
        "title": "Set up environment",
        "description": "Install Node.js, npm, and git",
        "completed": true
      },
      {
        "id": 2,
        "title": "Learn Express.js",
        "description": "Understand routing and middleware",
        "completed": false
      }
    ]
    ```
* **Example Response (200 OK - filtered by completed=true):**
    ```json
    [
      {
        "id": 1,
        "title": "Set up environment",
        "description": "Install Node.js, npm, and git",
        "completed": true
      }
    ]
    ```
* **Example Response (400 Bad Request - invalid `completed` value):**
    ```json
    {
      "message": "Invalid value for 'completed' query parameter. Must be 'true' or 'false'."
    }
    ```

---

### 2. Get Task by ID

* **Method:** `GET`
* **Path:** `/tasks/:id`
* **Description:** Retrieves a single task by its unique ID.
* **URL Parameters:**
    * `:id`: The unique integer ID of the task.
* **Example Request:**
    * `GET http://localhost:3000/tasks/1`
* **Example Response (200 OK):**
    ```json
    {
      "id": 1,
      "title": "Set up environment",
      "description": "Install Node.js, npm, and git",
      "completed": true
    }
    ```
* **Example Response (404 Not Found):**
    ```json
    {
      "message": "Task not found"
    }
    ```
* **Example Response (400 Bad Request - invalid ID format):**
    ```json
    {
      "message": "Invalid Task ID provided."
    }
    ```

---

### 3. Create a New Task

* **Method:** `POST`
* **Path:** `/tasks`
* **Description:** Creates a new task. The ID is automatically generated by the server.
* **Request Body (JSON):**
    * `title`: (Required, string) The title of the task. Must not be empty.
    * `description`: (Required, string) A detailed description of the task. Must not be empty.
    * `completed`: (Required, boolean) The completion status of the task. Must be `true` or `false`.
* **Example Request:**
    ```http
    POST http://localhost:3000/tasks
    Content-Type: application/json

    {
      "title": "Write README",
      "description": "Document all API endpoints and setup instructions.",
      "completed": false
    }
    ```
* **Example Response (201 Created):**
    ```json
    {
      "id": 3,
      "title": "Write README",
      "description": "Document all API endpoints and setup instructions.",
      "completed": false
    }
    ```
* **Example Response (400 Bad Request - validation errors):**
    ```json
    {
      "messages": [
        "Title is required and must be a non-empty string.",
        "Completed status is required and must be a boolean value."
      ]
    }
    ```

---

### 4. Update an Existing Task

* **Method:** `PUT`
* **Path:** `/tasks/:id`
* **Description:** Updates an existing task identified by its ID. Supports partial updates (you only need to send the fields you want to change).
* **URL Parameters:**
    * `:id`: The unique integer ID of the task to update.
* **Request Body (JSON):**
    * `title`: (Optional, string) New title for the task. Must not be empty.
    * `description`: (Optional, string) New description for the task. Must not be empty.
    * `completed`: (Optional, boolean) New completion status for the task. Must be `true` or `false`.
* **Example Request (Update `completed` status):**
    ```http
    PUT http://localhost:3000/tasks/1
    Content-Type: application/json

    {
      "completed": false
    }
    ```
* **Example Response (200 OK):**
    ```json
    {
      "message": "Task with ID 1 updated successfully."
    }
    ```
* **Example Request (Update title and description):**
    ```http
    PUT http://localhost:3000/tasks/2
    Content-Type: application/json

    {
      "title": "Refine Express.js Knowledge",
      "description": "Deep dive into advanced Express concepts like error handling middleware."
    }
    ```
* **Example Response (200 OK):**
    ```json
    {
      "message": "Task with ID 2 updated successfully."
    }
    ```
* **Example Response (404 Not Found):**
    ```json
    {
      "message": "Task not found"
    }
    ```
* **Example Response (400 Bad Request - invalid input/no fields provided):**
    ```json
    {
      "message": "No fields provided for update."
    }
    ```
    OR
    ```json
    {
      "messages": [
        "Title must be a non-empty string."
      ]
    }
    ```

---

### 5. Delete a Task

* **Method:** `DELETE`
* **Path:** `/tasks/:id`
* **Description:** Deletes a task by its unique ID.
* **URL Parameters:**
    * `:id`: The unique integer ID of the task to delete.
* **Example Request:**
    * `DELETE http://localhost:3000/tasks/1`
* **Example Response (204 No Content):**
    * A successful deletion will return a `204 No Content` status with an empty response body.
* **Example Response (404 Not Found):**
    ```json
    {
      "message": "Task not found"
    }
    ```
* **Example Response (400 Bad Request - invalid ID format):**
    ```json
    {
      "message": "Invalid Task ID provided."
    }
    ```
